④ 专家 Agent 池 · 四位工程师

真正动手写代码、跑测试、截图验证的执行层。

🔩 四个专家的本质：Prompt + LLM + 工具

每个 Expert 不是独立的程序，而是同一个 AI（Qwen2.5-Coder:7b）加上不同的「岗位说明书」。不同的只是系统 Prompt 和分配到的工具。

BackendExpert
角色：「你是后端工程师」
模型：Qwen2.5-Coder:7b
工具：写文件 / 运行代码
FrontendExpert
角色：「你是前端工程师」
模型：Qwen2.5-Coder:7b
工具：写文件
TestExpert
角色：「你是测试工程师」
模型：Qwen2.5-Coder:7b
工具：写文件 / 运行 pytest
UIValidator
角色：「你是UI验证工程师」
模型：Qwen2.5-Coder:7b
工具：Playwright 截图 / 点击
🗄️
BackendExpert
后端工程师 · 数据库代码
写让数据能存进去、取出来的代码。第一个跑，其他人都要等它完成才能开始。
1
收到任务：「写待办事项数据库，支持增删改查」
2
AI 生成 Python + SQLite 代码
3
调用「写文件」工具，存成 todo_db.py
4
运行一遍确认没语法错误
5
把文件内容写进 ProjectState，通知其他人可以开始了
# AI 真实生成并存到磁盘的文件 import sqlite3 def create_todo(title: str): conn = sqlite3.connect('todo.db') conn.execute( "INSERT INTO todos (title) VALUES (?)", (title,) ) conn.commit()
与 Frontend 并行 ✅ SQLite 按接口规范实现
🖥️
FrontendExpert
前端工程师 · 界面代码
写用户看得到的界面。采用接口优先设计，与 BackendExpert 并行开发——两者都读 Commander 生成的接口规范，不需要等对方完成。
1
从 ProjectState 读取接口规范（函数名、参数、返回值）
2
AI 生成 Tkinter 界面代码，按规范调用函数（Backend 还没写完也没关系）
3
存成 todo_app.py
4
写进 ProjectState
# 按接口规范调用，不依赖 Backend 的实现 import tkinter as tk from todo_db import create_todo, get_all_todos # ↑ 函数名来自 Commander 生成的规范 # Backend 和 Frontend 都按这份规范走 def on_add(): title = entry.get() create_todo(title) refresh_list()
与 Backend 并行 ✅ Tkinter 读接口规范
🧪
TestExpert
测试工程师 · 自动化测试
写自动化测试并真实运行，证明代码能用。测试结果直接影响 Validator 的判断。
1
从 ProjectState 读取 todo_db.py
2
AI 生成覆盖增删改查的 pytest 测试
3
存成 test_todo.py
4
真实运行 pytest，拿到结果
5
把「5 passed / 1 failed」写进 ProjectState
# AI 写的测试，真实运行 def test_create_todo(): create_todo("买牛奶") todos = get_all_todos() assert "买牛奶" in [t.title for t in todos] def test_delete_todo(): create_todo("临时任务") delete_todo("临时任务") todos = get_all_todos() assert "临时任务" not in [t.title for t in todos]
与 Frontend 并发 pytest 结果影响 Validator
📸
UIValidator ↗ Playwright
UI 验证工程师 · 截图 + 元素检查
真实启动应用，像人一样看界面，确认按钮存在、布局正确。这是最「真实」的验证——它真的打开了一个窗口。
1
等 FrontendExpert 完成（界面代码必须先有）
2
Playwright 启动应用，截图
3
检查「添加按钮」「任务列表」「删除按钮」是否存在
4
把截图路径和验证结果写进 ProjectState
# Playwright 像人一样操作界面 page.goto("http://localhost:3000") page.fill("#todo-input", "测试任务") page.click("#add-button") # 检查任务出现了 assert page.locator(".todo-item").count() > 0
真实启动应用 Playwright MCP 截图存档

📋 四个人靠 ProjectState 共享数据

他们不互相发消息，都通过这块「大白板」读写数据。每个人只写自己负责的部分。

ProjectState = { # Commander 写的 "requirement": "开发待办事项 App", "task_list": [backend, frontend, test, ui], # BackendExpert 写的 ↓ "todo_db.py": "import sqlite3\ndef create_todo...", # FrontendExpert 写的 ↓（先读了 todo_db.py） "todo_app.py": "import tkinter\nfrom todo_db import...", # TestExpert 写的 ↓ "test_todo.py": "def test_create_todo...", "test_results": "5 passed, 0 failed", # UIValidator 写的 ↓ "screenshot": "todo_app_screenshot.png", "ui_check_passed": True, "iteration_count": 1, }
