整个系统的大脑：把一句话需求拆解成任务、调度 Expert Agent、管理记忆。

🧠 先搞懂：LangGraph 是什么

普通代码是线性的，但 AI Agent 需要「判断 + 循环」：写完代码要检查，不通过要重写，通过了才结束。LangGraph 让你把这种流程画成图。

普通代码（线性）

第1步 → 第2步 → 第3步 → 完成

不能回头，不能循环

LangGraph（有条件、可循环）

分析 → 写代码 → 检查
              ↑          ↓ 通过 → 完成
              └── 修复 ←─┘ 不通过

每个格子叫 Node（节点），箭头叫 Edge（边），带条件的箭头叫条件边。所有节点共享一份数据叫 State（状态）。

class ProjectState(TypedDict): requirement: str # 你输入的需求 task_list: list # 拆解后的任务列表 generated_files: dict # 已生成的代码文件 iteration_count: int # 当前第几轮修复 validation_result:Optional # 最新验证结果 # ↑ 所有 Agent 共享这份数据，像一块大白板

记忆管理 ↗ mem0 · ⭐25k+
mem0 + ChromaDB + SQLite · 跨会话上下文
mem0 内部自带一个 LLM（本项目用 Ollama），专门负责判断哪些内容值得长期记住。存储分两处：ChromaDB 存向量（语义搜索用），SQLite 存对话流水账（写入时分析用）。
传入：「用了SQLite，第二轮通过，文件名todo_db.py，开始时间14:32」 ↓ mem0 内部 LLM（Ollama）分析 ① 提炼：「项目偏好 SQLite」✅ 「文件名/时间」❌ 没意义 ② 去重：已有「用户喜欢简洁界面」→ 新内容不重复 → 保留 ③ 存入：转向量 → ChromaDB；流水账 → SQLite

⚙️ 四个组件的完整配合流程

1
你输入「开发待办事项 App」→ 意图拆解引擎 调取 mem0 历史偏好，用 Qwen2.5:14b 生成 DAG 任务图
2
任务调度器 把任务放入 asyncio.Queue，BackendExpert 先跑，完成后 FrontendExpert / TestExpert / UIValidator 并发开始
3
每个 Expert 完成一步，审计与反思的 MemorySaver 自动存一个档，同时检查输出格式
4
所有 Expert 完成后，Validator 检查质量 → 不通过：条件边指回修复节点，最多 5 轮 → 通过：推送结果到前端
5
任务结束，记忆管理 把这次任务的结论存入 mem0，下次任务 Commander 更懂你
