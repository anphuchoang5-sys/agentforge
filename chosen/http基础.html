HTTP 基础 · GET / POST / DELETE 是什么

浏览器和服务器说话的「动作词」，搞懂它后端逻辑全通。

🗣️ 浏览器和服务器怎么「说话」

你每次打开网页、提交表单、点击按钮，浏览器都在和服务器对话，用的语言叫 HTTP。

HTTP 规定了几种「动作」，告诉服务器「我要做什么」——这就是 GET / POST / DELETE，统称 HTTP 方法。

🧑‍💻
你（前端）
浏览器里的网页
→
📋
HTTP 请求
GET / POST / DELETE
+ 地址 + 数据
→
🖥️
服务器（后端）
FastAPI

把服务器想象成一个图书馆，HTTP 方法是你和馆员说话的方式——

📚 四种 HTTP 方法（用图书馆比喻）

GET
📖 「我要借一本书」
只取数据，不改任何东西。就像去图书馆借书查书，书架上的书一本没少。
你：GET /api/tasks/001 馆员：{ id: "001", status: "完成", ... }
  • ✅ 打开网页
  • ✅ 查询订单状态
  • ✅ 搜索商品
POST
📦 「我要捐一本新书进来」
提交数据，创建新东西。必须带着「货物」（数据）一起发，服务器创建后返回新编号。
你：POST /api/tasks 附带：{ 需求: "做待办App" } 馆员：{ task_id: "task_001" } ← 新编号
  • ✅ 注册账号
  • ✅ 发评论
  • ✅ 提交 AI 任务（本项目）
DELETE
🗑️ 「把这本书销毁」
删掉某个已有的东西。指定编号，服务器把它删掉。
你：DELETE /api/tasks/001 馆员：{ message: "已删除" }
  • ✅ 删帖子
  • ✅ 取消订单
  • ✅ 停止 AI 任务（本项目）
PATCH
✏️ 「更新书里的某段内容」
修改已有的东西。PATCH 只改你指定的字段，PUT 是整个替换。
你：PATCH /api/tasks/001 附带：{ status: "paused" } 馆员：{ id: "001", status: "paused" }
  • ✅ 改头像昵称
  • ✅ 修改任务状态
  • ✅ 更新配置

🔗 URL 是什么（「去哪里」）

每次请求都要指定一个地址，告诉服务器「我要操作的是什么东西」：

http://localhost:8000 / api / tasks / 001 └── 服务器地址 ──────┘ └─分组─┘ └─资源─┘ └─具体哪个─┘

同一个地址，配不同的动作，完全不同的事情：

动作地址含义
GET/api/tasks取所有任务列表
GET/api/tasks/001取编号 001 的任务详情
POST/api/tasks新建一个任务
DELETE/api/tasks/001删除编号 001 的任务

📬 请求和响应（「寄件」和「收件」）

每次 HTTP 对话分两半，来回各一次：

→ 你发出去（Request 请求）

POST /api/tasks
Content-Type: application/json

{
  "需求": "开发待办事项App",
  "优先级": "高"
}

↑ 动作 + 地址
↑ 说明货物格式
↑ 货物本体（Body）
  只有 POST/PUT/PATCH 才有

← 服务器回来（Response 响应）

200 OK

{
  "task_id": "task_001",
  "status": "started",
  "created_at": "2026-07-01"
}

↑ 状态码（成功还是失败）
↑ 返回的数据

🚦 状态码（服务器的「表情」）

服务器用一个三位数字告诉你「这次对话结果怎么样」：

200
OK
成功
「给你」
201
Created
创建成功
「新书登记好了」
400
Bad Request
你发的格式不对
「你填的表有错」
404
Not Found
找不到
「没有这本书」
500
Server Error
服务器崩了
「图书馆着火了」

规律：2xx = 成功，4xx = 你的问题，5xx = 服务器的问题

⚙️ 回到这个项目 — 实际是怎么用的

# 第一步：提交任务（POST）
前端 → POST /api/tasks { "需求": "开发待办App" }
后端 → 201 { "task_id": "task_001" }

# 第二步：建立 WebSocket 实时连接
前端 → 连接 ws://localhost:8000/ws/tasks/task_001

# 第三步：后端主动推进度（不需要你 GET）
后端推送 → { agent: "Commander", msg: "拆解需求中..." }
后端推送 → { agent: "Backend",   msg: "生成代码中..." }
后端推送 → { agent: "Validator", msg: "✅ 验收通过" }

# 第四步：想停止任务（DELETE）
前端 → DELETE /api/tasks/task_001
后端 → 200 { "message": "已停止" }
