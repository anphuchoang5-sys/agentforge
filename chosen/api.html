② API 层 · FastAPI + WebSocket + Pydantic + SQLite

前端和后端之间的「通信管道」，以及数据格式检查和任务存储。

🍽️ 先搞懂：API 到底是什么？

API（应用程序接口）就是双方约定好的沟通规则。它和「AI 订阅」没有任何关系——只要两个程序需要互相传数据，它们之间就有 API。

最好懂的比喻：把系统想象成一家餐厅——

🧑 你（前端） 浏览器里的网页
⇄
👨‍💼 服务员（API） FastAPI 接口层
⇄
👨‍🍳 厨房（后端/AI） LangGraph + Ollama

你不能直接冲进厨房（前端不能直接操作数据库或调用 AI），只能对着菜单点菜（调用 API 的固定地址）。API 负责把你的请求传进去、把结果传出来。

📡 两种通信方式：REST 和 WebSocket

这个项目同时用了两种方式，各有用途：

✉️REST API — 像发短信

你发一条，它回一条，这次对话就结束了。适合「提交任务」「查询结果」这类一问一答的场景。

前端：POST /api/tasks
       { "需求": "做待办App" }

后端：{ "task_id": "task_001",
       "status": "started" }

↑ 完事，这次连接关闭

📞WebSocket — 像打电话

连接建立后一直保持，后端随时可以主动推消息给前端。适合「AI 干活过程中实时显示进度」。

前端：连接 /ws/tasks/task_001

后端：「Commander 开始分析了」
后端：「BackendExpert 写代码中」
后端：「第1轮验证不通过，修复」
后端：「✅ 完成！代码已生成」

↑ 前端从没主动问过一次

xterm.js 实时滚动日志、流程图节点变色，都是靠 WebSocket 推过来的。

· 这一层的 4 个组件 ·
⚡
FastAPI ↗ GitHub · ⭐80k+
Python 后端框架，提供 REST 接口
Python 里最快的 Web 框架，原生支持异步（不会因为等 AI 响应而卡死）。自动生成接口文档，开发时可以直接在浏览器里测试每个接口。
POST /api/tasks ← 提交新需求 GET /api/tasks/{id} ← 查询任务状态 DELETE /api/tasks/{id} ← 停止任务 GET /api/agents ← 查看所有 Agent
原生异步 自动生成文档 内置 Pydantic
🔌
WebSocket
实时双向通信，FastAPI 内置支持
建立一条「一直开着的管道」。AI 每完成一个步骤，后端立刻通过这条管道把消息推给前端——不需要前端不停地问「好了没」。
连接地址：ws://localhost:8000/ws/tasks/task_001 后端每步推送一条 JSON： { "agent": "BackendExpert", "status": "running", "message": "正在生成 todo_db.py...", "iteration": 1 }
后端主动推送 零延迟更新 持久连接
🔍
Pydantic v2 ↗ GitHub · ⭐22k+
数据格式检查员
规定数据必须长什么样，不符合格式直接报错拦截。对 Agent 系统特别重要——因为 LLM 有时会输出格式混乱的文本，Pydantic 帮你确保拿到的数据是可以程序化处理的。
class SubTask(BaseModel): id: str # 必须是字符串 type: str # 必须是规定的类型 description: str # 不能为空 # AI 返回格式乱了 → 直接报错 # 不让坏数据继续往下走 ✅
格式验证 Rust 实现超快 AI 结构化输出必备
🗄️
SQLAlchemy + SQLite ↗ GitHub
任务历史记录存储
SQLite = 一个超轻量数据库，就是一个 .db 文件，不需要安装任何服务。
SQLAlchemy = 让你用写 Python 的方式操作数据库，不用手写 SQL 语句。
# 不用写 SQL：SELECT * FROM tasks WHERE id='001' # 直接写 Python： task = session.query(Task).filter( Task.id == 'task_001' ).first() # 存：谁 / 什么时候 / 写了什么 / 迭代几轮
零配置 任务历史 Agent 状态记录

⚙️ 四个组件怎么配合工作

1
你在前端提交需求 → FastAPI REST 接收请求，给这个任务分配一个 ID
2
Pydantic v2 检查请求格式——描述不能为空、类型必须合法，不通过直接拒绝
3
SQLite 把任务记录存进数据库，哪怕服务器重启，历史记录还在
4
AI Agent 开始工作，每走一步就通过 WebSocket 把进度推给前端
5
前端的 xterm.js 实时滚动、流程图节点实时变色 → 你看到的全程直播效果就是这么来的
